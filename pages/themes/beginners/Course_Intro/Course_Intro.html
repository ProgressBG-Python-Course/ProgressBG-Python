Python Virtual Environments with Pipenv

Virtual environment - What and Why?

Many system modules depends on some specific versions of Python's packages - it's not a good idea to pollute your global python environment
Virtual environments create an isolated Python environment for your project
Keep workflows between projects independent.
Keeps dependency management hell away
There many tools for working with virtual environments in Python

Pipenv - What?

A dependency manager for Python projects
Automatically creates and manages a virtualenv for your projects
Functionally similar to npm and other modern package managers.
It’s the recommended higher-level tool to simplify dependency management
Combines the use of pip and virtualenv

Pipenv - Install


			pip install --user pipenv

Pipenv - manage virtualenv


				### Create virtualenv with system Python3:
				$ pipenv --three
				Creating a virtualenv for this project…


				### Spawn a shell within the virtualenv
				$ pipenv shell
				Spawning environment shell...				

				### Exit the virtualenv
				(course_intro-SmgljMaj) $ exit
				exit
				$ 

				### Output virtualenv information
				$ pipenv --venv

				### Remove the virtualenv
				$ pipenv --venv
				Removing virtualenv...

Pipenv - manage packages for your project

		
			### install package
			$ pipenv install requests
			Installing requests…

			### remove package
			pipenv uninstall requests
			Uninstalling requests-2.18.4

Check the installed packages


			pip freeze
